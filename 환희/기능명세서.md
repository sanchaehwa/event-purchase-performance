# 로그인 / 회원가입 기능 명세서

- 담당자: 김환희
- 기술 스택: React

---

## 1. 로그인 페이지 (`/login`)

### 화면 구성

| 요소 | 설명 |
|------|------|
| 이메일 입력 필드 | type="email" |
| 비밀번호 입력 필드 | type="password", 표시/숨김 토글 버튼 포함 |
| 자동 로그인 체크박스 | 체크 시 Refresh Token 만료 기간 30일로 연장 |
| 로그인 버튼 | 이메일 + 비밀번호 입력 시 활성화 |
| 비밀번호 찾기 링크 | `/forgot-password` 로 이동 |
| 카카오 로그인 버튼 | 카카오 OAuth2 연동 |
| 구글 로그인 버튼 | 구글 OAuth2 연동 |
| 회원가입 링크 | `/signup` 으로 이동 |

### 기능 상세

**이메일 로그인**
1. 이메일, 비밀번호 입력 후 로그인 버튼 클릭
2. 클라이언트 유효성 검사 (이메일 형식, 비밀번호 공백 여부)
3. `POST /api/auth/login` 요청 (자동 로그인 여부 포함)
4. 성공 시 Access Token 저장 후 홈(`/`)으로 이동
5. 실패 시 입력 필드 하단에 에러 메시지 표시

**소셜 로그인**
1. 카카오/구글 버튼 클릭 시 각 OAuth2 인가 페이지로 리다이렉트
2. 인증 완료 후 서버에서 JWT 발급 → 홈(`/`)으로 이동

### 유효성 검사 (클라이언트)

| 필드 | 조건 | 에러 메시지 |
|------|------|-------------|
| 이메일 | 이메일 형식 | 올바른 이메일 형식을 입력해주세요 |
| 비밀번호 | 공백 불가 | 비밀번호를 입력해주세요 |

### 서버 에러 처리

| 상황 | 표시 메시지 |
|------|-------------|
| 이메일/비밀번호 불일치 | 이메일 또는 비밀번호가 올바르지 않습니다 |
| 이메일 미인증 계정 | 이메일 인증이 필요합니다 |
| 서버 오류 | 잠시 후 다시 시도해주세요 |

### 페이지 이동

| 조건 | 이동 경로 |
|------|-----------|
| 로그인 성공 | `/` (홈) |
| 비밀번호 찾기 링크 클릭 | `/forgot-password` |
| 회원가입 링크 클릭 | `/signup` |
| 이미 로그인된 상태로 접근 | `/` (홈)으로 리다이렉트 |

## 2. 회원가입 페이지 (`/signup`)

회원가입은 **3단계 스텝** 으로 구성됩니다.

```
[1단계] 이메일 입력 → [2단계] 인증 코드 확인 → [3단계] 정보 입력 + 약관 동의 → 완료
```

---

### 1단계 - 이메일 입력

**화면 구성**

| 요소 | 설명 |
|------|------|
| 이메일 입력 필드 | type="email" |
| 인증 메일 발송 버튼 | 이메일 형식 맞을 때 활성화 |
| 카카오로 시작하기 버튼 | 카카오 OAuth2 |
| 구글로 시작하기 버튼 | 구글 OAuth2 |
| 로그인 링크 | `/login` 으로 이동 |

**기능**
1. 이메일 입력 후 인증 메일 발송 버튼 클릭
2. `POST /api/auth/signup` 요청
3. 성공 시 2단계로 전환
4. 이메일 중복 시 에러 메시지 표시

**유효성 검사**

| 필드 | 조건 | 에러 메시지 |
|------|------|-------------|
| 이메일 | 이메일 형식 | 올바른 이메일 형식을 입력해주세요 |
| 이메일 | 중복 없음 (서버) | 이미 사용 중인 이메일입니다 |

---

### 2단계 - 이메일 인증 코드 확인

**화면 구성**

| 요소 | 설명 |
|------|------|
| 안내 문구 | `{이메일}로 발송된 인증 코드를 입력해주세요` |
| 인증 코드 입력 필드 | 6자리 숫자, 카운트다운 타이머 표시 (5:00) |
| 확인 버튼 | 6자리 입력 시 활성화 |
| 재발송 버튼 | 타이머 만료 후 활성화 |

**기능**
1. 6자리 코드 입력 후 확인 버튼 클릭
2. `POST /api/auth/email/verify` 요청
3. 성공 시 3단계로 전환
4. 실패/만료 시 에러 메시지 표시

**에러 처리**

| 상황 | 표시 메시지 |
|------|-------------|
| 코드 불일치 | 인증 코드가 올바르지 않습니다 |
| 코드 만료 | 인증 코드가 만료되었습니다. 재발송해주세요 |

---

### 3단계 - 정보 입력 + 약관 동의

**화면 구성**

| 요소 | 설명 |
|------|------|
| 이름 입력 필드 | 최대 20자 |
| 비밀번호 입력 필드 | type="password", 표시/숨김 토글 |
| 비밀번호 확인 필드 | type="password" |
| 전체 동의 체크박스 | 아래 항목 일괄 선택/해제 |
| 이용약관 동의 체크박스 | 필수, 약관 내용 보기 링크 포함 |
| 개인정보 처리방침 동의 체크박스 | 필수, 내용 보기 링크 포함 |
| 마케팅 수신 동의 체크박스 | 선택 |
| 가입 완료 버튼 | 필수 항목 모두 충족 시 활성화 |

**기능**
1. 이름, 비밀번호, 비밀번호 확인 입력
2. 필수 약관 2개 동의 체크
3. 가입 완료 버튼 클릭
4. 클라이언트 유효성 검사 통과 시 `POST /api/auth/signup/complete` 요청
5. 성공 시 로그인 페이지(`/login`)로 이동 + 완료 토스트 메시지 표시

**유효성 검사**

| 필드 | 조건 | 에러 메시지 |
|------|------|-------------|
| 이름 | 1~20자 | 이름을 입력해주세요 |
| 비밀번호 | 8자 이상, 영문+숫자+특수문자 포함 | 비밀번호는 8자 이상, 영문/숫자/특수문자를 포함해야 합니다 |
| 비밀번호 확인 | 비밀번호 일치 | 비밀번호가 일치하지 않습니다 |
| 이용약관 | 필수 동의 | 이용약관에 동의해주세요 |
| 개인정보 처리방침 | 필수 동의 | 개인정보 처리방침에 동의해주세요 |

### 페이지 이동

| 조건 | 이동 경로 |
|------|-----------|
| 소셜 가입/로그인 성공 | `/` (홈) |
| 이메일 가입 완료 | `/login` |
| 이미 로그인된 상태로 접근 | `/` (홈)으로 리다이렉트 |
| 로그인 링크 클릭 | `/login` |

## 3. 비밀번호 찾기 페이지 (`/forgot-password`)

비밀번호 찾기는 **2단계** 로 구성됩니다.

```
[1단계] 이메일 입력 → [2단계] 새 비밀번호 설정 → 완료
```

---

### 1단계 - 이메일 입력

**화면 구성**

| 요소 | 설명 |
|------|------|
| 이메일 입력 필드 | type="email" |
| 인증 메일 발송 버튼 | 이메일 형식 맞을 때 활성화 |
| 로그인으로 돌아가기 링크 | `/login` 으로 이동 |

**기능**
1. 이메일 입력 후 인증 메일 발송 버튼 클릭
2. `POST /api/auth/password/reset-request` 요청
3. 성공 시 2단계로 전환
4. 가입되지 않은 이메일이어도 보안상 동일한 안내 메시지 표시 ("인증 메일이 발송되었습니다")

---

### 2단계 - 새 비밀번호 설정

**화면 구성**

| 요소 | 설명 |
|------|------|
| 인증 코드 입력 필드 | 6자리 숫자, 카운트다운 타이머 표시 (5:00) |
| 새 비밀번호 입력 필드 | type="password", 표시/숨김 토글 |
| 새 비밀번호 확인 필드 | type="password" |
| 변경 완료 버튼 | 모든 필드 유효 시 활성화 |

**기능**
1. 인증 코드 + 새 비밀번호 입력
2. `POST /api/auth/password/reset` 요청
3. 성공 시 로그인 페이지(`/login`)로 이동 + 완료 토스트 메시지 표시

**유효성 검사**

| 필드 | 조건 | 에러 메시지 |
|------|------|-------------|
| 인증 코드 | 6자리 숫자 | 인증 코드를 입력해주세요 |
| 새 비밀번호 | 8자 이상, 영문+숫자+특수문자 포함 | 비밀번호는 8자 이상, 영문/숫자/특수문자를 포함해야 합니다 |
| 새 비밀번호 확인 | 비밀번호 일치 | 비밀번호가 일치하지 않습니다 |

**에러 처리**

| 상황 | 표시 메시지 |
|------|-------------|
| 인증 코드 불일치 | 인증 코드가 올바르지 않습니다 |
| 인증 코드 만료 | 인증 코드가 만료되었습니다. 재발송해주세요 |
| 서버 오류 | 잠시 후 다시 시도해주세요 |

### 페이지 이동

| 조건 | 이동 경로 |
|------|-----------|
| 비밀번호 변경 완료 | `/login` |
| 로그인으로 돌아가기 클릭 | `/login` |

## 4. API 명세서

| 메서드 | 엔드포인트 | 설명 | 인증 필요 |
| :--- | :--- | :--- | :---: |
| POST | `/api/auth/login` | 이메일 로그인 (자동 로그인 옵션 포함) | X |
| POST | `/api/auth/signup` | 회원가입 인증 메일 발송 | X |
| POST | `/api/auth/email/verify` | 이메일 인증 코드 확인 | X |
| POST | `/api/auth/signup/complete` | 회원가입 완료 (최종 정보 입력 및 약관 동의) | X |
| POST | `/api/auth/password/reset-request` | 비밀번호 재설정 메일 발송 요청 | X |
| POST | `/api/auth/password/reset` | 인증 코드 검증 및 새 비밀번호 설정 | X |
| POST | `/api/auth/logout` | 로그아웃 (토큰 무효화) | O |
| GET | `/api/auth/me` | 현재 로그인한 사용자 정보 조회 (상태 유지용) | O |
| POST | `/api/auth/refresh` | Access Token 갱신 (Refresh Token 사용) | X |
